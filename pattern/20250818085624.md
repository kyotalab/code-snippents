## 再設計の落とし穴

### Anti Pattern

スクラッチ再開発による問題解決

- **再設計自体が自信過剰の現れ**
- 元のプロジェクトと同様の崩壊パターンを繰り返す
- 根本的な設計思考の改善なしに成功はない

---

### 実例分析

`hawk` プロジェクトの構造問題

```bash
src/
├── arg.rs      # 引数処理
├── error.rs    # エラー処理
├── executor.rs # 実行ロジック
├── filter.rs   # フィルタ機能
├── main.rs     # エントリポイント
└── ...         # その他ユーティリティ
```

#### 問題点

- レイヤー分離の欠如
- 責務の不明確な分散
- ドメインロジックとインフラの混在
